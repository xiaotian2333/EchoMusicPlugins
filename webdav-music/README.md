# WebDAV 音乐

连接 WebDAV 服务器，浏览和播放云端音乐文件。

## 功能

- **🎵 云端音乐浏览**：通过 WebDAV 协议浏览远程服务器上的音乐文件，支持目录导航和面包屑路径
- **▶️ 播放控制**：支持双击播放单曲、播放全部、点击文件夹播放按钮一键播放整个文件夹
- **🏷️ 元数据提取**：自动读取音频文件的嵌入标签（ID3v2 / FLAC），获取标题、歌手、专辑、封面图和歌词
- **🖼️ 封面展示**：优先读取嵌入封面，其次探测目录下的封面文件（cover.jpg、folder.jpg 等），兜底跟随主应用默认封面
- **📁 文件夹管理**：支持多层目录结构，文件夹与歌曲混合排序显示
- **🔐 认证支持**：支持 Basic Auth 用户名/密码认证

## 安装

将插件文件夹 `webdav-music` 放入 EchoMusic 的插件目录中即可。

```
EchoMusicPlugins/
└── webdav-music/
    ├── manifest.json
    ├── icon.svg
    ├── index.js
    └── style.css
```

## 使用

1. 在插件设置中填写 WebDAV 服务器地址、用户名、密码和根目录路径
2. 点击左侧边栏的「WebDAV」进入浏览页面
3. 双击歌曲即可播放，双击文件夹进入子目录
4. 点击文件夹左侧的播放按钮可直接播放文件夹内全部音乐

## 设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|
| 服务器地址 | WebDAV 服务器 URL，如 `https://webdav.example.com` | - |
| 用户名 | Basic Auth 用户名，留空表示无需认证 | - |
| 密码 | Basic Auth 密码 | - |
| 根目录路径 | 浏览音乐的起始路径 | `/` |
| 在线匹配封面歌词 | 开启后使用酷狗搜索出的封面和歌词，关闭则优先使用歌曲内嵌的封面和歌词 | 关闭 |

## 支持的音频格式

`mp3` `flac` `wav` `aac` `ogg` `wma` `m4a` `ape` `opus` `aiff` `alac` `dsf` `dff`

## 歌曲命名格式

插件通过文件名解析歌曲标题和歌手，支持以下命名格式（按优先级匹配）：

| 格式 | 示例 | 解析结果 |
|------|------|----------|
| `歌手 - 标题` | `周杰伦 - 晴天.mp3` | 歌手：周杰伦，标题：晴天 |
| `歌手 – 标题`（短横线 en-dash） | `Taylor Swift – Shake It Off.mp3` | 歌手：Taylor Swift，标题：Shake It Off |
| `歌手 — 标题`（长横线 em-dash） | `Adele — Hello.flac` | 歌手：Adele，标题：Hello |
| `编号. 歌手 - 标题` | `01. 林俊杰 - 江南.mp3` | 歌手：林俊杰，标题：江南 |
| `编号- 歌手 - 标题` | `02-陈奕迅-十年.mp3` | 歌手：陈奕迅，标题：十年 |

> 编号前缀为行首的 `数字` 后跟 `.`、`-` 或空格，解析时会自动去除。
> 如果文件名不符合上述格式，则整个文件名（不含扩展名）作为标题，歌手显示为「未知歌手」。
> 若音频文件包含内嵌标签（ID3v2 / FLAC），解析后会以嵌入标签为准覆盖文件名解析结果。

## 封面与歌词获取

插件支持两种封面和歌词获取模式，可在插件设置中切换：

### 关闭「在线匹配封面歌词」（默认）

优先使用歌曲内嵌数据，缺少时通过酷狗 API 补充：

1. 音频文件内嵌封面和歌词（ID3v2 / FLAC 标签）
2. 目录下的封面图片文件（cover.jpg、folder.jpg 等）
3. 酷狗 API 搜索补全缺失的封面或歌词
4. 主应用兜底封面（跟随 `cover-fallback` 插件设置动态切换）

### 开启「在线匹配封面歌词」

优先使用酷狗搜索结果，不保证封面和歌词的准确性：

1. 酷狗 API 搜索结果中的封面和歌词
2. 音频文件内嵌数据补充酷狗未提供的字段
3. 主应用兜底封面（跟随 `cover-fallback` 插件设置动态切换）
